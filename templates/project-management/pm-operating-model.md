# PM Operating Model

## 目标
把项目推进、多人协同和交付闭环拆成清晰主线，而不是把所有过程都混进业务知识页。

## 3 个固定板块
1. Portfolio / 项目总览
2. Team coordination / 团队协同
3. Personal execution / 个人执行

## 固定边界
- 业务事实 → 原业务 area
- 全库治理 → governance
- PM 方法 / 看板 / gate / register → project-management

## 默认状态
- artifact：`draft / approved / blocked / superseded`
- project：`active / blocked / done / parked`

## 路由规则
- 多人事项 → `knowledge-base-team-coordination`
- 稳定知识回写 → `knowledge-base-maintenance`
- 验收 / greenlight → `knowledge-base-delivery-audit`
