Why this package exists

The goal is not to help you record more notes. It is to stop a vault from decaying into a maze of folders, dead links, mixed page roles, and hard-to-reuse knowledge.

Typical drifting vault Target state with this package
Root pages become timeline dumps full of temporary execution history. Root pages return to navigation, stable boundaries, and topic entrypoints.
ops pages become chronological logs of what happened once. ops pages become reusable symptom / root cause / fix / boundary references.
Long-form sources are imported as one big blob and become difficult to revisit. Sources are split into overview / chapter / topic structures with lineage and better retrieval surfaces.
Collaboration context sits in chat, meeting notes, and personal memory. Working profile, team coordination, and journal workflows create a stable, reviewable collaboration layer.

What’s inside

The repository keeps the package explicit: 8 public skills, one fixed page-role model, and a small set of repeatable workflows rather than an opaque automation black box.

onboarding

knowledge-base-kit-guide

Explains installation, profile setup, and which skill to use next.

orchestration

knowledge-base-orchestrator

Provides low-friction onboarding: inspect the current environment, create a skeleton, generate a profile, and route the next step.

ingest

knowledge-base-ingest

Imports long-form markdown with a baseline-first, test-driven, lightweight-harness workflow.

maintenance

knowledge-base-maintenance

Writes durable conclusions back into the right page role while filtering one-off process noise.

audit

knowledge-base-audit

Inspects metadata, dead links, orphan pages, boundary drift, and noise regression.

working profile

knowledge-base-working-profile

Distills collaboration-relevant preferences, heuristics, and boundaries without turning them into a private dossier.

team coordination

knowledge-base-team-coordination

Coordinates shared-project questionnaires, alignment, assignments, and decision distillation.

work journal

work-journal

Captures daily work, meeting notes, and temporary ideas that can later feed durable knowledge.

One method, several high-density scenarios

This package is especially strong in environments that are knowledge-dense, frequently updated, collaboration-heavy, and in need of traceability and auditability.

Medical / pathology / research

Guidelines, textbooks, papers, and case summaries need structure, lineage, and careful update boundaries.

  • Long-form source ingest
  • Stable method and ops pages
  • Review-friendly collaboration context

Enterprise knowledge bases

PRDs, runbooks, delivery docs, and meeting conclusions need reliable navigation and governance instead of ad hoc sprawl.

  • Maintenance and audit loops
  • Shared-project coordination
  • Cleaner onboarding for teammates

Cross-team collaboration

When context is fragmented across chat and temporary docs, shared coordination and journal workflows help prevent repeated re-explanation.

  • Draft / approved boundaries
  • Traceable meeting and decision context
  • Working-profile hygiene with consent boundaries

How it flows

Start with onboarding, stabilize the profile contract, then move through specialist loops that keep the vault usable as it grows.

01

Install

Copy all 8 skill folders into the skills directory of your AI platform.

02

Choose onboarding

Use Orchestrator for a low-friction start or Kit Guide for an explanation-first path.

03

Prepare profile

Create or refine vault-profile.md so later workflows have a stable contract.

04

Run specialist loops

Move into ingest, maintenance, audit, working-profile, team-coordination, or work-journal as needed.

05

Keep it stable

Repeat governance and navigation checks so the vault stays useful instead of drifting again.

Fixed page-role model

The package stays stable because page boundaries are explicit. It does not rely on every contributor improvising structure from scratch.

fixed roles

5 durable page roles

  • project for navigation, overview, and boundaries
  • knowledge for reusable methods and concepts
  • ops for troubleshooting and procedures
  • task for assignments and in-progress work
  • overview for indexes and governance surfaces
boundary note

Journal is not a sixth role

work-journal remains a separate workflow. Journal content is allowed to be more fluid, but durable knowledge should be promoted into the fixed page-role model instead of staying in the journal forever.
