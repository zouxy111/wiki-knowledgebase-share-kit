Why this package exists

The point is not to write more notes. The point is to stop a vault from decaying into a maze of folders, mixed page roles, broken navigation, and hard-to-reuse knowledge.

Typical drifting vault Target state with this package
Root pages become timeline dumps full of temporary execution history. Root pages return to navigation, stable boundaries, and topic entrypoints.
ops pages become chronological logs of what happened once. ops pages become reusable symptom / root cause / fix / boundary references.
Long-form sources are imported as one big blob and become difficult to revisit. Sources are split into overview / chapter / topic structures with lineage and retrieval surfaces.
Collaboration context sits in chat, meeting notes, and personal memory. Working profile, team coordination, and journal workflows create a stable, reviewable collaboration layer.

What’s inside

The repository keeps the package explicit: 10 public skills, one fixed page-role model, and a small set of repeatable workflows rather than a black-box automation promise.

onboarding

knowledge-base-kit-guide

Explains installation, profile setup, and which skill to use next.

orchestration

knowledge-base-orchestrator

Provides low-friction onboarding: inspect the environment, create a skeleton, generate a profile, and route the next step.

ingest

knowledge-base-ingest

Imports long-form markdown with a baseline-first, test-driven, lightweight-harness workflow.

maintenance

knowledge-base-maintenance

Writes durable conclusions back into the right page role while filtering one-off process noise.

audit

knowledge-base-audit

Inspects metadata, dead links, orphan pages, boundary drift, and noise regression.

project management

knowledge-base-project-management

Builds owner-side summaries, milestones, blockers, and personal execution boards without mixing management surfaces into governance.

team coordination

knowledge-base-team-coordination

Coordinates questionnaires, alignment, assignments, and closeout decision distillation.

delivery audit

knowledge-base-delivery-audit

Checks delivery completeness, evidence gaps, write-back gaps, and ready / blocked / greenlight conditions.

working profile

knowledge-base-working-profile

Distills collaboration-relevant preferences, heuristics, and boundaries without turning them into a private dossier.

work journal

work-journal

Captures daily work, meeting notes, and temporary ideas that can later feed durable knowledge.

How it flows

Start with onboarding, stabilize the profile contract, then move through specialist loops that keep the vault usable as it grows.

01

Install

Copy all 10 skill folders into the skills directory of your AI platform.

02

Choose onboarding

Use Orchestrator for a low-friction start or Kit Guide for an explanation-first path.

03

Prepare profile

Create or refine vault-profile.md so later workflows have a stable contract.

04

Run specialist loops

Move into ingest, maintenance, audit, and only load project-management, team-coordination, or delivery-audit when those workflows are actually needed.

05

Keep it stable

Repeat governance and navigation checks so the vault stays useful instead of drifting again.

fixed model

5 durable page roles

  • project for navigation, overview, and boundaries
  • knowledge for reusable methods and concepts
  • ops for troubleshooting and procedures
  • task for assignments and in-progress work
  • overview for indexes and governance surfaces
boundary note

Journal is not a sixth role

Journal content can be more fluid, but durable knowledge should be promoted into the fixed page-role model instead of remaining in the journal forever.
