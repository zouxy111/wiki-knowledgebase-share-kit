8-skill knowledge-base package

Keep your markdown vault useful, not noisy.

A reusable open-source kit with 8 installable skills and 7 coordinated workflow tracks: onboarding, ingest, maintenance, audit, working profile, team coordination, and work journaling. Optional Obsidian installation exists through Orchestrator, but the repository’s core value is stable knowledge-base workflows—not being an app installer.

30-second outcome

This repository is for vaults that have drifted into a mix of project logs, unstable navigation, and hard-to-reuse notes.

Typical drifting vault Target state with this package
Root pages accumulate temporary execution history. Root pages return to navigation, stable boundaries, and topic entrypoints.
ops pages become chronological logs. ops pages become reusable symptom / root cause / fix / boundary references.
log.md turns into task replay. log.md stays milestone-only.
New teammates or future-you do not know where to start. Onboarding and specialist skills provide a stable routing path.

What’s inside

The package is meant to be explicit: 8 public skills, one fixed page-role model, and a small number of repeatable workflows.

knowledge-base-kit-guide

Explains installation, profile setup, and which skill to use next.

knowledge-base-orchestrator

Optional onboarding coordinator: check current environment, create a skeleton, generate a profile, and route the user forward.

knowledge-base-ingest

Import long-form markdown with a baseline-first, test-driven, lightweight-harness mindset.

knowledge-base-maintenance

Write durable conclusions back into the right page role while filtering process noise.

knowledge-base-audit

Inspect metadata, dead links, orphan pages, boundary drift, and noise regression.

knowledge-base-working-profile

Distill stable collaboration-relevant preferences, heuristics, and boundaries without turning them into a private dossier.

knowledge-base-team-coordination

Coordinate shared-project questionnaires, alignment, assignments, and closeout decision distillation.

work-journal

Capture daily work, meeting notes, temporary ideas, and weekly summaries that can feed durable knowledge later.

How it flows

Use Orchestrator when you want the easiest start, Kit Guide when you want to understand the model first, then move into specialist loops.

01InstallCopy all 8 skill folders into your AI platform’s skills/ directory.
02Choose onboardingUse knowledge-base-orchestrator for low-friction setup or knowledge-base-kit-guide for a more explicit explanation-first path.
03ProfileCreate or refine your vault-profile.md so later specialist skills have a stable contract.
04Specialist loopsMove into ingest, maintenance, audit, working-profile, team-coordination, or work-journal depending on the task.
05StabilizeKeep navigation, page roles, and durable knowledge aligned as the vault grows.

Test-driven ingest example

A good long-form ingest is not a one-shot import. The initial structure should be treated as a baseline, then improved through testing, regression checks, version comparison, and lightweight harness-style iteration.

Baseline first

Import the source into an overview/chapter/topic structure with lineage, TOC, glossary candidates, and related-link suggestions.

Compare and refine

Compare split strategies, page roles, and link architecture using real retrieval and maintenance feedback.

Refactor safely

Run regression checks on entry pages, lineage, and key links after each revision so the vault becomes cleaner instead of noisier.
