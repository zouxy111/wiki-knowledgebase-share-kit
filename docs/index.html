Core package
5

Guide, ingest, maintenance, audit, and team-coordination skills designed to stay reusable across vaults and shared project folders.

Method constraints

Stable page roles, ingest promotion by rubric, OpenClaw / Hermes as recommended collaboration runtimes, shared project directory as source of truth, and draft → approved gating for tasks and decisions.

30-second outcome

This project is for vaults and markdown-based collaborations that have drifted into logs, unstable structure, and inconsistent coordination. The kit pulls them back toward durable retrieval and reusable workflows.

Typical drifting state Target state with this kit
Root pages accumulate temporary execution history. Root pages return to navigation, stable boundaries, and topic entrypoints.
ops pages become chronological logs. ops pages become reusable symptom / root-cause / fix / boundary references.
Long-form ingest gets rewritten ad hoc. Imports start as a baseline, compare candidates with a rubric, and only promote post-regression winners.
Multi-person work lives only in chat. Shared project directories hold questionnaires, alignment, assignments, follow-up, and decision distillation.
Everyone works privately with no stable convergence point. Personal wiki + personal agent are allowed, but synced markdown in the shared project directory becomes the formal input.

What’s inside

The kit now covers both durable knowledge-base work and multi-person project coordination without turning either workflow into an unstructured prompt pile.

Guide

Installation, template setup, first-run routing, and deciding whether the user should start with ingest, maintenance, audit, or team coordination.

Ingest

Split long markdown sources into reusable page groups, produce TOC / glossary / related-link artifacts, and compare candidate structures against a stable baseline.

Maintenance

Write durable conclusions into the right page role, filter process noise, and sync content plus navigation surfaces.

Audit

Check metadata, dead links, orphan pages, boundary drift, navigation coverage, and noise regression.

Team coordination

Run intake, questionnaires, alignment, assignment, follow-up, and decision distillation for 2+ people, with optional vault sync for approved stable outcomes.

How it flows

01InstallCopy the five skill folders into your AI platform’s skills/ directory.
02Profile / WorkspaceFill a vault profile for knowledge-base work, or prepare a shared project directory for team coordination.
03IngestStart with a stable baseline, create a candidate structure for one improvement goal, and evaluate it with a shared rubric.
04Maintain / CoordinateWrite durable conclusions into the vault, or drive multi-person questionnaires, alignment, assignments, and follow-up from the shared directory.
05Audit / PromoteRun regression checks, inspect drift, and only promote changes or decisions once they meet the required standards.

Recommended collaboration setup

OpenClaw / Hermes

Recommended runtimes for the coordination workflow. Compatibility comes from the shared-project-directory contract plus the skills / prompts model, not from a private API.

NAS / cloud drive or shared wiki

Keep team-project/ either as its own synced folder or as a subtree inside a shared wiki / shared knowledge base. The directory remains the formal fact source in both modes.

Personal wiki + synced facts

Each member can draft in their own wiki or private workspace with their own agent. Only markdown synced back into the shared project directory enters the official coordination loop.
