# Member Profile Model

## Dual-layer model

### A. Cross-project reusable profile
记录长期稳定画像：
- identity / display name
- strengths
- weaknesses / support needed
- preferred task granularity
- preferred output formats
- typical decision areas
- recurring constraints
- collaboration style
- reusable notes from past projects

### B. Project-local member context
记录本项目临时信息：
- this-project role
- current materials in hand
- available time / bandwidth
- current goal understanding
- current blockers

## Canonical profile path rule
如果 `team-roster.md` 提供了 `canonical profile path`：
- 先读取该画像
- 再结合当前项目资料生成问卷

如果没有提供：
- 不应失败
- 用第一轮问卷建立项目内画像
- 只在反复观察到稳定特征时，才把它升级成可复用画像增量

## What counts as reusable
- 长期稳定的强项 / 约束 / 偏好
- 重复出现的任务边界
- 对问卷颗粒度的稳定偏好
- 典型协作接口

## What should stay project-local
- 本次项目的临时时间安排
- 本次手头资料
- 本次范围变化
- 临时 blocker
