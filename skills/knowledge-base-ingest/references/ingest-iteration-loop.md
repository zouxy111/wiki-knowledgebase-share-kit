# Ingest Iteration Loop

长篇知识源导入不应被视为一次性动作，而应被视为一个**稳定基线 → 候选结构 → 回归检查 → 晋升决策**的循环。

## 默认循环

1. 先确定当前 stable baseline  
2. 每轮只集中解决一种主要结构问题  
3. 生成 candidate structure  
4. 用同一 rubric 比较 baseline 与 candidate  
5. 跑最小回归检查  
6. 通过则晋升；否则继续打磨或放弃本轮候选结构  

## 为什么每轮只改一类问题

如果一轮同时改：
- 拆分粒度
- 页面角色
- 导航结构
- 关联链接

那后续很难判断到底是哪种变化真正带来了改进。

因此更推荐：
- 一轮只优先调一个主问题
- 把其他变化压到最低
- 保证结果可归因、可比较、可回退

## 最小回归检查

至少检查：
- overview 入口是否仍清楚
- root page / reader entrypoint 是否同步
- source lineage 是否完整
- 有无新增死链和孤立页
- 候选结构是否比当前基线更利于检索和维护

## 决策口径

- **promote**：候选结构成为新基线
- **rework**：候选结构方向对，但还没达到晋升标准
- **drop**：这轮候选结构不值得继续保留
