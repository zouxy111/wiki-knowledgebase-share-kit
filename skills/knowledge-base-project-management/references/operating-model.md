# Project Management Operating Model

## Position in the package
`knowledge-base-project-management` 是核心 share kit 里的 **PM 主线入口**。

它负责：
- 项目 intake
- owner 视角拆解
- 优先级 / 里程碑 / blocker / dependency
- 本周推进板
- PM 层 routing

它不负责：
- 代替多人协同问卷与 alignment
- 代替知识库 maintenance
- 代替交付验收 gate 审计

## Management-layer area rule
`project-management` 只放：
- portfolio board
- personal execution board
- PM operating model
- delivery gates
- risk register
- decision register

不要放：
- 业务知识正文
- 业务运维细节
- 原始长文档正文
- 需要回原业务 area 的稳定业务事实

## Three-board model
### A. Portfolio
看项目组合、阶段、优先级、成功标准。

### B. Team coordination
多人事项进入 `knowledge-base-team-coordination` 的 shared project workflow。

### C. Personal execution
看本周推进、blocker、dependency、next action。
