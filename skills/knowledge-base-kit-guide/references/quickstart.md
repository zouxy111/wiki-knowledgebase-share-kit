# Quickstart

## 最短上手路径

1. 先读 share kit 根目录下的 `README.md`
2. 再读 `docs/customization-guide.md`
3. 复制 `templates/vault-profile-template.md`
4. 填好你自己的 vault 路径、areas、root pages、frontmatter 规则
5. 安装：
   - `skills/knowledge-base-maintenance`
   - `skills/knowledge-base-audit`
6. 第一次落地时：
   - 先用 maintenance 沉淀一个小结果
   - 再用 audit 看结构是否健康

## 如果用户只想先看懂
推荐顺序：
- `README.md`
- `docs/usage-sop.md`
- `examples/example-vault-profile.md`

## 如果用户已经安装但不会用
先确认：
- 有没有填写 `vault profile`
- 是否明确 root pages
- 是否明确 canonical root-level markdown files
- 是否知道 maintenance 和 audit 的职责边界
