# Quickstart

## 最短上手路径

1. 先读 share kit 根目录下的 `README.md`
2. 再读 `START-HERE.md`
3. 如果是知识库场景，复制 `templates/vault-profile-template.md`
4. 如果是多人协同场景，复制 `templates/team-project-workspace/`
5. 如需成员画像复用，复制 `templates/member-capability-profile-template.md`
6. 安装：
   - `skills/knowledge-base-kit-guide`
   - `skills/knowledge-base-maintenance`
   - `skills/knowledge-base-audit`
   - `skills/knowledge-base-team-coordination`
7. 第一次调用时，显式告诉 agent：
   - profile 文件的**绝对路径**
   - 或共享项目目录的**绝对路径**

## 如果用户只想先看懂
推荐顺序：
- `README.md`
- `docs/reuse-from-zero.md`
- `docs/team-coordination-workflow.md`
- `docs/usage-sop.md`
- `examples/example-vault-profile.md`
- `examples/team-project-generic/README.md`

## 如果用户已经安装但不会用
先确认：
- 有没有填写 `vault profile`
- 是否明确 root pages
- 是否明确 canonical root-level markdown files
- 是否知道 maintenance、audit、team-coordination 的职责边界
- 有没有准备共享项目目录
- `team-roster.md` 是否能定位成员目录和 canonical profile 路径
