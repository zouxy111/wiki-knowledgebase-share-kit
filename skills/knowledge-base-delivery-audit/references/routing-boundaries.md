# Routing Boundaries

## Use delivery-audit when
- 要判交付是否闭环
- 要做 closeout 前审查
- 要决定 ready / blocked / greenlight pending

## Route back to project-management when
- 发现缺口其实是优先级或 next-action 问题
- 还需要重新整理 owner 视角的推进节奏

## Route to team-coordination when
- 缺口来自多人未对齐
- assignment 需要重排
- 需要补问或重新收集成员响应

## Route to maintenance when
- 审计确认已形成稳定结论
- 需要把 approved 结果回写业务 area
